Política de Privacidade

Última atualização: junho de 2026 — Versão 1.0

O Mercuri é uma plataforma de atendimento via WhatsApp. Atuamos como operadores de dados em nome das clínicas que utilizam nosso sistema. Esta política explica como tratamos os dados pessoais processados pela plataforma.

1. Quem somos

O Mercuri é desenvolvido e operado por [Nome do desenvolvedor / empresa], CNPJ [número]. Para fins da LGPD (Lei nº 13.709/2018), atuamos como operadores dos dados pessoais, processando informações em nome das clínicas e revendedores que contratam nossos serviços (os controladores).

As clínicas que utilizam o Mercuri são as controladoras dos dados de seus pacientes e são responsáveis pelas decisões sobre o tratamento dessas informações.

2. Quais dados coletamos

Dados dos pacientes (via WhatsApp)

  • Nome completo
  • Número de telefone (WhatsApp)
  • Informações de saúde fornecidas voluntariamente durante o atendimento (especialidade desejada, tipo de exame, convênio)
  • Histórico de conversas com a clínica

Dados dos agentes e supervisores

  • Nome e endereço de e-mail
  • Senha (armazenada exclusivamente como hash bcrypt — nunca em texto simples)
  • Registro de acessos e ações no sistema (logs de auditoria)

3. Para que usamos os dados

  • Prestação do serviço de atendimento automatizado e humano via WhatsApp
  • Roteamento de mensagens para a equipe correta da clínica
  • Geração de resumos de atendimento para os agentes por IA
  • Relatórios gerenciais para supervisores das clínicas
  • Segurança e melhoria contínua do sistema

Não vendemos, compartilhamos ou utilizamos dados pessoais de pacientes para fins de marketing ou publicidade.

4. Base legal para o tratamento

O tratamento de dados pessoais é realizado com base nos seguintes fundamentos da LGPD:

  • Art. 7º, V — Execução de contrato: para prestação do serviço contratado pela clínica
  • Art. 7º, II — Cumprimento de obrigação legal ou regulatória
  • Art. 11, II, f — Exercício regular de direitos, para os dados sensíveis de saúde

6. Segurança dos dados

  • Criptografia TLS em trânsito e em repouso no banco de dados
  • Isolamento técnico entre clínicas (Row-Level Security no PostgreSQL)
  • Controle de acesso baseado em funções — agentes veem apenas sua fila
  • Registros de auditoria imutáveis de todas as ações no sistema
  • Senhas armazenadas exclusivamente como hash bcrypt (irreversível)
  • Rate limiting nas rotas de autenticação para prevenir ataques de força bruta

7. Retenção e exclusão de dados

Os dados são mantidos pelo período necessário à prestação do serviço e por até 5 anos após o encerramento do contrato, para cumprimento de obrigações legais. Após esse prazo, os dados são excluídos de forma segura e irreversível.

A clínica controladora pode solicitar a exclusão antecipada dos dados de seus pacientes a qualquer momento, mediante solicitação formal por e-mail.

8. Direitos dos titulares

Os pacientes (titulares dos dados) podem exercer os seguintes direitos previstos na LGPD, mediante solicitação à clínica responsável pelo atendimento:

  • Confirmação da existência do tratamento
  • Acesso aos dados
  • Correção de dados incompletos ou incorretos
  • Eliminação dos dados
  • Portabilidade dos dados
  • Informação sobre compartilhamento com terceiros
  • Revogação do consentimento
